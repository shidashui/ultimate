# Agents（智能体系统架构）

## 系统架构

```
┌──────────────────────────────────────────────┐
│  主系统（全局统筹者）                          │
│  Global Decision & Execution System           │
│  职责：目标分解与分配 · 全局状态监控 · 协调     │
├──────────────────────────────────────────────┤
│  子系统（Sub-Agent 集群）                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐      │
│  │ Sub A    │ │ Sub B    │ │ Sub C    │ ...  │
│  │ workspace│ │ workspace│ │ workspace│      │
│  │ session  │ │ session  │ │ session  │      │
│  │ memory   │ │ memory   │ │ memory   │      │
│  └──────────┘ └──────────┘ └──────────┘      │
└──────────────────────────────────────────────┘
```

## 主系统职责

- 维持全局目标状态与进度追踪
- 将任务分解为可分配给子系统（或自身执行）的子任务
- 监控各子系统执行状态
- 在子系统间协调数据流
- 检测全局异常并启动对应预案

## 子系统隔离规则

- 每个子系统拥有独立的 workspace、会话记录和记忆存储
- 路由绑定决定每条消息由哪个子系统处理
- 子系统之间互相隔离：一个子系统无法读取其他子系统的会话或记忆
- 子系统之间不直接通信

## 系统间协调（Coordination）

系统之间的协调通过以下机制实现：
1. **共享工作空间文件**：通过文件系统进行数据交换（若已配置）
2. **路由层调度**：网关层决定消息分发到哪个子系统
3. **人工操作者**：人类通过网关 API 或 CLI 进行跨系统管理
